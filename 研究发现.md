# 研究发现

## 2026-04-17 UART DMA + IDLE 接收改造

- 当前项目平台为 STM32F407 + HAL + CMSIS-RTOS2/FreeRTOS。
- `USART1` 已在 `Core/Src/usart.c` 中配置为 `115200 8N1`，并已打开 `DMA2_Stream2` 作为 RX DMA。
- `USART1_IRQn` 和 `DMA2_Stream2_IRQn` 已在生成代码中启用，`stm32f4xx_it.c` 中也已经正确转发给 HAL。
- 当前 HAL 版本在 `Drivers/STM32F4xx_HAL_Driver/Inc/stm32f4xx_hal_uart.h` 和 `Src/stm32f4xx_hal_uart.c` 中已提供：
  - `HAL_UARTEx_ReceiveToIdle_DMA()`
  - `HAL_UARTEx_RxEventCallback()`
- 因此本轮不需要手写寄存器级 IDLE 检测逻辑，直接采用 HAL 官方的 DMA + 空闲中断接收路径最稳妥。
- 用户需求已从“周期主动发送重量”收敛为“电脑发送查询命令时才返回当前重量”。
- 本轮协议计划采用最小文本命令集：`GET` 查询当前重量；未知命令返回错误提示。
