# LDC1614 电感检测串口通知任务

## Goal
在当前 STM32F407 FreeRTOS 工程中新增 LDC1614 电感检测功能，使用硬件 I2C 读取模块数据，并在检测目标放上去或移开后，自动通过串口发送一次状态消息。

## Requirements
- 基于 `ldc1614电感检测模块/` 目录中的资料与例程实现，不凭空猜测寄存器配置。
- 优先采用硬件 I2C，而不是模拟 I2C。
- 用户代码必须放在 `User/` 下：
  - 驱动层放在 `User/Driver`
  - 应用层放在 `User/App`
- 需要与现有 FreeRTOS 工程兼容，不能把复杂业务逻辑塞进 `Core` 生成文件。
- 串口输出沿用现有 `USART1` 输出路径，满足线程安全要求。
- 检测逻辑需要具备“变化完成后只发送一次消息”的行为，避免持续刷屏。
- 代码必须补充中文函数头注释和关键逻辑注释。

## Acceptance Criteria
- [ ] 工程中新增 LDC1614 驱动模块，能够完成设备初始化、寄存器读写和通道数据读取。
- [ ] 工程中新增 LDC1614 应用任务，能够完成基线采样、滤波、变化检测和稳定判定。
- [ ] 当目标放上去导致检测值稳定变化后，串口自动发送一次“检测到变化”消息。
- [ ] 当目标移开并恢复稳定后，串口自动发送一次“恢复/移除”消息。
- [ ] 工程启用硬件 I2C 外设并完成引脚配置，同步更新工程配置文件。
- [ ] 新增源码已加入 Keil 工程，并符合当前项目目录规范和注释规范。

## Technical Notes
- 当前工程空闲且适合复用的硬件 I2C 方案优先考虑 `I2C2 + PB10/PB11`，与资料中的“硬件 IIC2”例程保持一致，降低移植偏差。
- 检测任务应与现有重量任务解耦，避免共享外设冲突。
- 初版默认以 `CH0` 作为检测通道，阈值、稳定次数和采样周期做成可配置宏。
