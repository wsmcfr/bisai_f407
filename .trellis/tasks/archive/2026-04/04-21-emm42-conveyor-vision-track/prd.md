# Emm42 步进电机传送带视觉跟踪任务

## Goal
在当前 STM32F407 FreeRTOS 工程中新增基于张大头 `Emm42_V5.0` 闭环步进电机的传送带控制功能。电机通过 `USART2` 与主控通信，在“主机坐标使能标志位=0”时低速匀速运行，在“主机坐标使能标志位=1”时根据主机发送的目标坐标偏差动态调速，并在目标到达摄像头中心附近时立即停止。

## Requirements
- 基于 `HAL_Emm_TTL/STM32F407_串口通讯__速度模式/` 例程开展集成，优先复用其 `Emm_V5.c/.h` 封装，不重复造轮子。
- 用户代码必须放在 `User/` 下：
  - 驱动层放在 `User/Driver`
  - 应用层放在 `User/App`
- 默认主控制模式采用“速度模式”，不以“使能信号控制”作为主流程。
- 后续如果末端停车精度不足，可扩展“位置模式”作为补充精调方案，但不作为当前第一阶段主路径。
- 电机通信串口使用当前已经启用的 `USART2`，避免和现有 `USART1` 调试/命令口混用。
- 需要与当前 FreeRTOS 工程兼容，任务设计必须满足线程安全、实时性和资源可控性要求。
- 状态机至少覆盖：
  - 巡航状态：坐标未使能时低速匀速转动
  - 跟踪状态：坐标使能后按误差调节速度
  - 停止状态：目标进入中心死区且满足稳定判定后立即停止
- 串口与电机资源归属必须清晰，不能出现多个任务无序抢占同一串口发送控制命令。
- 代码必须写中文函数头注释、关键变量注释和关键状态切换注释。
- 本轮先完成任务定义和技术路线，后续实现阶段不主动编译，由用户提供编译结果。

## Acceptance Criteria
- [ ] 工程中新增 Emm42 驱动封装模块，能够完成基础命令发送与必要状态控制接口封装。
- [ ] 工程中新增传送带电机应用服务/任务，完成巡航、视觉跟踪、中心停止三态控制。
- [ ] 明确主机侧输入接口格式，包括使能标志位、坐标偏差或中心误差的接入方式。
- [ ] 明确速度控制映射策略，包括最小巡航速度、跟踪速度上限、中心死区与稳定判定条件。
- [ ] 明确 `USART2` 在当前工程中的资源归属和与其它串口模块的边界。
- [ ] 新增源码及后续相关工程改动符合当前目录规范、注释规范与 FreeRTOS 并发规范。

## Technical Notes
- 当前阶段优先研究并迁移 `HAL_Emm_TTL/STM32F407_串口通讯__速度模式/` 中的接口，再决定哪些最小命令集需要落到本工程。
- 推荐控制路线：
  - 粗跟踪采用速度模式
  - 误差进入死区后直接停机
  - 若后续需要更高重复定位精度，再考虑补充位置模式做末端微调
- “主机坐标使能标志位”在当前理解中是软件状态位，不等同于电机硬件 `EN` 引脚控制。
- 串口命令建议统一由单一应用入口下发，驱动层只负责格式化并发送电机控制帧。
- 由于当前仓库仍有未提交变更，后续实施时必须避免误覆盖现有 LDC1614 与工程配置相关改动。
